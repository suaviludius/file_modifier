Алгоритм работы программы:

1. КОНФИГУРАЦИЯ

а) маска входных файлов, (например, *.txt, *.bin)
d) путь для поиска in файлов
c) путь для сохранения out файлов
f) режим работы (по таймеру / разовый запуск)
b) настройка необходимости удалять входные файлы или нет
g) периодический опрос (таймер)
e) действия при конфликте имен (перезапись / счетчик)
h) 8-битное значение для XOR в HEX

2. ПОИСК ФАЙЛА
- сбор списка файлов для обработки

3. ФАЙЛ
- проверка на возможность чтения (доступен / занят)
- создание файла результата (решение конфликта имен)
- обработка (с возможностью паузы / отмены)
    - открыть in файл (бинарный режим)
    - открыть out файл (бинарный режим)
    - читать блоками (для 10Гб - 4Мб)
    - применить XOR для каждого блока
    - записать обработанный блок в out файл
    - обновлять прогресс
    - проверять флаг остановки/паузы на каждой итерации
    - сохранять состояние для возобновления
- close файлы
- обработать результат по статусу
    - ПАУЗА - сохранить позицию
    - ОТМЕНА - удалить частичный выходной файл
    - ЗАВЕРШЕНО - удалить исходный файл (если настроено)

4. ЛОАД БАР
- текущий обрабатываемый файла
-прогресс в процентах
- скорость обработки
- оставшееся время (дополнительный)

5. ОБРАБОТЧИК ОСТАНОВКИ
- остановка ВСЕХ потоков (или сделать RAII обертку и не переживать)
- сохранение состояний незавершенных файлов
- корректное закрытие и очищение рессурсов (проверить санитайзером и профайлером)
