Класс для работы с файлом

Использование:
Для запуска процессинга, будем создавать в бекенде объекты процнссинга
для каждого потока отдельно и заупскать в общем пуле QThreadPool.

План обработки файла:
+ Открыть IN файл в бинарном режиме - QFile
+ открыть OUT файл в бинарном режиме в выбранной папке m_outputFile
   (с тем же именем или с добавлением номера)
+ Получить размер фала из метаданных
+ Проверить наличие сохраненного состояния (если была нажата пауза).
  Нужно иметь возможность приостановить и возобновить (если можно без сохранения состояния, то гуд, но пока хз как)
   Можно сохранять позицию внутри файла или сохранять отдельный файл, в который записывать состояние (например, .state)
5) В цикле читать блоки (например 1Мб) из входного файла, применять XOR к каждым 8 байтам, записывать в выходной
6) Обновить прогресс после обработки чанка
7) Проверить флаг остановки/паузы (атомарные флаги - всему голова, связывваемся с бекендом)
8) Настройки. Если стоял флаг на удаление исходного файла, то удаляем
9) Исключения. При ошибке или остановке удалять частичный файл или нет?
   Создавать разные файлы для возобновления после СТОП и ПАУЗА?


Контролировать выполнение разных этапов обработки файла
будем с помощью конечного автомата:

Состояния:
IDLE        - ожидание запуска
OPENING     -проверка и открытие файлов
PROCESSING  - основной цикл
PAUSED      - обработка приостановлена
STOPPING    - сигнал остановки, завершаем работу
COMPLETED    - успешное завершение
ERROR       -ошибка

Переходы:
- IDLE -> OPENING
- OPENING -> PROCESSING
- OPENING -> ERROR
- PROCESSING -> PAUSED
- PROCESSING -> STOPPING
- PROCESSING -> COMPLETED
- PROCESSING -> ERROR
- PAUSED -> PROCESSING
- PAUSED -> STOPPING
- PAUSED -> IDLE
- STOPPING -> ERROR


Для паузы и озобновления:
- при паузе просто проверяем флаг и ждём, пока не будет снята пауза
- при остановке прерываем обработку и уадаляем неполный файл
