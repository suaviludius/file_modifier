Алгоритм работы программы:

GUI <-> Бекенд <-> Процессор

1. GUI

КОНФИГУРАЦИЯ
+ маска входных файлов, (например, *.txt, *.bin)
+ путь для поиска in файлов
+ путь для сохранения out файлов
+ режим работы (по таймеру / разовый запуск) + таймер (таймер)
+ настройка необходимости удалять входные файлы или нет
- действия при конфликте имен (перезапись / счетчик)
+ 8-битное значение для XOR в HEX

ВАЛИДАЦИЯ
+ проверка наличия текста в текстовых полях
+ проверка валидности путей к папкам
+ проверка числа символов в HEX
+ проверка валидности символов в HEX с помощью regex

При нажатии кнопки НАЧАТЬ довыанные конфигурации валидируются
и передаются в бекенд.

2. BACKEND

ПОИСК ФАЙЛА
+ обработка сигналов управления НАЧАЛО / ПАУЗА-ПРОДОЛЖИТЬ / СТОП
+ чтение конфигурации из бекенда
+ сбор списка файлов для обработки (в in папке по маске)
+ автообновление списка файлов (режим по таймеру)
+ передача файла в процессор
+ обновление статус бара по возврату от бекенда

3. PROCESSOR

ФАЙЛ
+ проверка на возможность чтения (доступен / занят)
+ создание файла результата (решение конфликта имен)
+ открыть in файл (бинарный режим)
+ открыть out файл (бинарный режим)
+ читать блоками (для 10Гб - 1Мб)
+ применить XOR для каждого блока
+ записать обработанный блок в out файл
+ обновлять прогресс
+ проверять флаг остановки/паузы на каждой итерации
+ сохранять состояние для возобновления
+ close файлы
+- обработать результат по статусу
    - ПАУЗА - сохранить позицию
    - ОТМЕНА - удалить частичный выходной файл
    - ЗАВЕРШЕНО - удалить исходный файл (если настроено)
